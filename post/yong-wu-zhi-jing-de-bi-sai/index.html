永无止境的比赛

2022-09-27 算法题

题目描述
每年有数十场斯诺克比赛，Jinotega团队试图参加所有比赛（出于某种原因，他们更喜欢“斯诺克”这个名字）！当比赛在远离家乡的地方举行时，伊万、阿尔乔姆和康斯坦丁会乘飞机去参加比赛，然后再回来。

吉诺特加最好的朋友，基地团队已经找到了他们的行程单列表，其中包含有关出发和到达机场的信息。现在他们想知道，吉诺特加现在在哪里：在家里还是在遥远的比赛中？他们知道：

此列表包含吉诺特加今年的所有航班（按任意顺序排列），
吉诺特加只从家乡飞到斯诺克比赛，然后又回来了，
每次比赛结束后，吉诺特加都会飞回家（尽管他们可能会多次在一个地方参加比赛），
最后，在年初，吉诺特加在家。
请帮助他们确定吉诺特加的位置！

输入描述
在第一行输入中，有一个整数n：吉诺特加的航班数（1≤n≤100）。在第二行中，有一串3个大写的拉丁字母：吉诺特加家乡机场的名称。在接下来的n行中有航班信息，每行一个航班，形式为“XXX->YYY”，其中“XXX”是出发机场的名称“YYY”是到达机场的名称。这些机场正好是吉诺特加的家乡机场。

保证航班信息与吉诺特加朋友的知识一致，这在声明的主要部分有所描述。

输出描述
如果吉诺特加现在在家，打印“home”（不带引号），否则打印“contest”。

样例输入
4
SVO
SVO->CDG
LHR->SVO
SVO->LHR
CDG->SVO
样例输出
home
提示
在样本中，吉诺特加可能首先从SVO飞到CDG机场并返回，然后从SVO飞到LHR再返回，所以现在他们应该在家里

#include<stdio.h>
int main()
{
	char op[110];
	int n,m;
	scanf("%d",&n);
	m=n;
	getchar();
	gets(op);
	while(n--)
	{
		gets(op);
	}
	if(m%2==0)
	printf("home");
	else
	printf("contest");
	return 0;
}

为什么这道题读起来不是很通顺呢，因为他原题是全英文，这是经过浏览器翻译后的文本233.。。
8过问题不大，因为这是个脑筋急转弯题。
因为题目说了每次参加完比赛后都要回家，哪怕是在一个地方参加两次比赛，而且这个人的航班全部和斯诺克比赛有关，意思是只要航程次数为偶数，他最后就必定在家，如果为奇数他就必定在比赛。

其实如果题目换一下，每次比赛完后不一定直接回家，可能直接飞往下一次比赛场地，那么如何解呢？
或许可以写一个结构体，里面包含着出发地和目的地，再加一个参数在判断这个人的行程时判断该航班有没有判断过，然后再从头遍历结构体，得出完整的的航行路线，但这样时间复杂度是很大的，因为每次都遍历了整个结构体。

其实这样的话还有个问题就是输入的问题，例如 SVO->CDG,SVO和CDG是两个参数，那么再读入字符串时如何把它拆分呢？最笨的方法就是读整个字符串再由->作为分界点来拆分成两个字符串，但这未免也太麻烦了，不过除此之外还真想不到其他办法了。

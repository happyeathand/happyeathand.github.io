AcWing790. 数的三次方根

2022-05-07 算法基础课学习记录

给定一个浮点数 n，求它的三次方根。

输入格式
共一行，包含一个浮点数 n。

输出格式
共一行，包含一个浮点数，表示问题的解。

注意，结果保留 6 位小数。

数据范围
−10000≤n≤10000
输入样例：
1000.00
输出样例：
10.000000

解

#include<bits/stdc++.h>
using namespace std;

int main()
{
	double n,l,r,mid;
	cin>>n;
	l=-10000;
	r=10000;
	while(r-l>1e-8)/*浮点数的二分，结果可能是小数（应该说大多数情况都是小数）所以没法像整数二分那样又l=r的情况，所以判断条件写成r-l>1e-8,至少要写到1e-7因为要保留六位小数，写太多会超时，写太少通过不了*/
	{
		mid=l+(r-l)/2;
		if(mid*mid*mid>=n) r=mid;
		else l=mid;  /*为了精确所以不能写成l=mid+1，结果可能是小数，+1就直接跨过正确答案了*/
	}
	printf("%lf\n",l);/*冷知识，printt("%lf");自动保留6位小数，结尾输出l和r都是对的，我想原因是答案判断到后8位小数已经很精确了吧，所以两个数在保留小数时就一样了*/
}

浮点数的二分和整数的二分差别在两点
1.循环结束的条件不同，整数二分l和r到最后一定会重合，在l向右靠近时经过+1处理最后也会重合，所以最后输出l和r结果是一样的，循环结束的条件写成while(r>l)。然而浮点型二分在l和r分别向右或左靠拢时也靠不到对方，因为在循环里它们等于的都是mid，没有自增过，在两数贴近时通过一个精度比较高的判断while((r-l)>1e-8)，得到一个尽量比较精确的答案
2.自增，整数二分里mid与x比较后l或r是有自增的，依次达到l与r重合的目的，然而浮点二分里没有，因为没有最小单位可增，增小了会超时，增大了会与正确答案相距甚远。（个人理解）
